# 目前需要优化的地方

按照优先级排序

- 按说任务合成agent需要根据任务种子（config）搜索到一条能作为ground truth的东西，找不到的话需要不断调整搜索关键词，一直尝试搜索直到找到或者超出最大尝试次数认定这个种子不可行。但当前只是搜索固定次数，也没有尝试不同关键词的迭代过程。不管搜到的内容怎么样就传到下一步生成任务了，造成了很多ground truth不能用于评测
- 任务种子合成的时候需要带上具体的企业，比如“你要对XXX（一个具体的公司）进行企业调研”，而不是对“某公司”调研。这个选择的过程需要作为agent的一个步骤，可能要先搜索然后根据搜到哪些gt决定，可能要一开始生成种子数据的时候就决定，但当前这个是缺失的。
- GT和参考资料目前是同时出来的，搜出来内容选定最相关的一条作为GT，其他作为任务参考资料（data room），这样是否不合理？是否应该让 agent 自己决定是否要加入参考资料、加入什么样的参考资料、搜索什么关键词？
- 前期过于依赖 vibe coding 积累的技术债。比如只告诉codex任务中不许出现涉及训练的内容（因为没有那个资源去运行），codex不会从整个流程入手防止出现，而是在 agent.py 创建了一个 _enforce_sop_compliance() 函数把任务说明中的“训练”字样改成“推理”。其他地方应该也有技术债。
